

# 📌 Guía rápida: MailHog con Docker

## 🔹 Encender MailHog

1. Arrancar Docker (si está apagado):

```bash
sudo systemctl start docker
```

2. Iniciar el contenedor MailHog (si ya existe):

```bash
docker start mailhog
```

3. Si el contenedor no existe todavía, créalo:

```bash
docker run -d --name mailhog \
  -p 1025:1025 -p 8025:8025 \
  mailhog/mailhog
```

📍 Acceso:

* SMTP → `localhost:1025`
* Web UI → [http://localhost:8025](http://localhost:8025)

---

## 🔹 Apagar MailHog

1. Detener solo el contenedor:

```bash
docker stop mailhog
```

2. Apagar todo Docker (si no usarás más contenedores):

```bash
sudo systemctl stop docker
```

3. Eliminar el contenedor (opcional, si quieres limpiar):

```bash
docker rm mailhog
```

---

## 🔹 Arranque automático de Docker

* Ver si Docker se inicia con el sistema:

```bash
systemctl is-enabled docker
```

* **Deshabilitar inicio automático** (no arranca con el sistema):

```bash
sudo systemctl disable docker
```

* **Habilitar inicio automático** (arranca con el sistema):

```bash
sudo systemctl enable docker

# 📌 Guía rápida: Instalar y usar MailHog con Docker (Ubuntu)

## 🔹 1. Instalar Docker

```bash
sudo apt update
sudo apt install -y docker.io
```

## 🔹 2. Habilitar Docker

Arrancar y dejarlo activo:

```bash
sudo systemctl enable --now docker
```

(Con esto ya queda listo y se levanta cada vez que inicie el sistema, si lo desea puede deshabilitarlo luego con `sudo systemctl disable docker`).

## 🔹 3. Probar Docker

```bash
docker run --rm hello-world
```

👉 Si ves el mensaje de “Hello from Docker!”, está todo bien.

## 🔹 4. Descargar y levantar MailHog

```bash
docker run -d --name mailhog \
  -p 1025:1025 -p 8025:8025 \
  mailhog/mailhog
```

* **SMTP**: `localhost:1025`
* **Interfaz Web**: [http://localhost:8025](http://localhost:8025)

## 🔹 5. Usar MailHog en tu proyecto

En la configuración de correo (Jakarta Mail, PHP, Python, etc.):

```
host: 127.0.0.1
port: 1025
auth: false
tls: false
```

## 🔹 Solo apagar el contenedor MailHog

```bash
docker stop mailhog

## 🔹 Apagar Docker (todo el servicio)

```

## 🔹 Deshabilitar inicio automático de Docker

```bash
sudo systemctl disable docker
```

## 🔹 Arrancar manualmente Docker cuando lo necesite
